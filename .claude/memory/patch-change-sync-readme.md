凡是对 patch-claude skill 的补丁做任何改动——新增补丁、删除补丁、归档补丁、修改补丁的标题 / 效果 / 状态——都必须**同步更新** [README.md](README.md) 与 [README_cn.md](README_cn.md) 的「补丁列表」表格（位于 `## Features` / `## 功能` 一节），让两份 README 始终反映 `patches/` 目录的真实清单。

**Why:** README 的补丁列表是「这套 skill 到底改了扩展的哪些行为」的权威索引，是用户和未来的自己快速了解全貌的入口。若只改 `patches/*.md` 而漏更 README，README 会像 2026-07-18 这次一样停在「Six patches / 六个补丁」、漏掉 007–011，读者会误以为只有 6 个补丁。编号跳号（如 006 归档导致 005 之后直接到 007）也必须在表里体现或加脚注说明，否则读者会困惑编号为什么对不上。

**How to apply:**
- **新增补丁**：表格加一行（编号 / 标题 / 效果），中英两份 README 都要加。
- **删除 / 归档补丁**：从表里移除该行；若导致编号跳号，加脚注说明去向（如「006 已归档，故编号自 005 跳至 007」）。
- **改补丁标题或效果**：同步改表格对应行文字。
- **尚未实测通过 / 仍在原型验证的补丁**：在效果栏标注「（原型验证中 / 待实测）」，实测通过、正式落盘为补丁文件后去掉该标注。
- **中英两份必须一致更新**，不能只改一份。
- 这条规则目前靠人工遵守（commit skill 的项目标配检测未覆盖 README 补丁列表时效性）；每次动补丁时主动检查两份 README 的表格是否与 `patches/` 目录一致。
